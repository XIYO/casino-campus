# 📚 4일 완성 카드 게임 학습 플랜

## 🎯 학습 목표
Java의 객체지향 프로그래밍(OOP) 개념을 활용하여 카드 게임의 기본 구성 요소를 구현합니다.

## 📅 일정별 학습 계획

### Day 1: Card 구현 (카드 한 장 만들기)
**목표**: Card 인터페이스를 구현하여 카드 한 장을 표현할 수 있게 만들기

#### 오전 (2-3시간)
1. **프로젝트 구조 이해하기**
   - `src/card/` 폴더의 파일들 살펴보기
   - Card 인터페이스, Suit, Rank enum 이해하기

2. **Card 구현체 작성**
   - PlayingCard 클래스 생성
   - 필수 메서드 구현:
     - 생성자 (Suit, Rank를 받아서 초기화)
     - getSuit(), getRank(), getValue()
     - toString() (예: "♠A", "♥10")

#### 오후 (2-3시간)
3. **equals()와 hashCode() 구현**
   - 같은 무늬와 숫자를 가진 카드는 동일하게 처리
   - HashSet에서 중복 제거가 되도록 구현

4. **테스트 실행**
   - CardTest.java의 주석 해제하고 테스트
   - 모든 테스트가 통과할 때까지 수정

**💡 팁**: 
- 카드는 불변(immutable) 객체로 만들기 (final 필드 사용)
- toString()에서 무늬 기호(♠♥♦♣)를 사용하면 보기 좋음

---

### Day 2: Deck 구현 (카드 덱 만들기)
**목표**: 52장의 카드를 관리하는 Deck 구현하기

#### 오전 (2-3시간)
1. **Deck 구현체 작성**
   - StandardDeck 클래스 생성
   - 내부에 List<Card>로 카드들 관리
   - 생성자에서 52장의 카드 생성

2. **기본 메서드 구현**
   - drawCard(): 맨 위 카드 뽑기
   - getRemainingCards(), isEmpty()
   - reset(): 52장으로 초기화

#### 오후 (2-3시간)
3. **shuffle() 구현**
   - Collections.shuffle() 활용
   - 무작위로 카드 순서 섞기

4. **추가 메서드 구현**
   - drawCards(int count): 여러 장 뽑기
   - contains(Card card): 특정 카드 확인

5. **테스트 실행**
   - DeckTest.java 테스트 통과시키기
   - 예외 처리 확인 (빈 덱에서 뽑기 등)

**💡 팁**:
- ArrayList를 사용하면 편리함
- 카드를 뽑을 때는 리스트에서 실제로 제거해야 함
- 예외 상황을 꼼꼼히 처리하기

---

### Day 3: Hand 구현 (손패 만들기)
**목표**: 플레이어의 손에 든 카드들을 관리하는 Hand 구현하기

#### 오전 (2-3시간)
1. **Hand 구현체 작성**
   - PlayerHand 클래스 생성
   - 내부에 List<Card>로 손패 관리
   - addCard(), removeCard() 구현

2. **카드 관리 메서드 구현**
   - getCards(): 방어적 복사본 반환
   - contains(), clear()
   - getCardCount(), isEmpty()

#### 오후 (2-3시간)
3. **sort() 메서드 구현**
   - Comparator 활용하여 정렬
   - 무늬 순서: SPADE < HEART < DIAMOND < CLUB
   - 같은 무늬 내에서는 숫자 순서

4. **테스트 실행**
   - HandTest.java 테스트 통과시키기
   - 경계 조건 테스트 (빈 손패, null 처리 등)

**💡 팁**:
- getCards()는 새로운 리스트를 반환해야 함 (방어적 복사)
- Comparator.comparing()을 체이닝하면 정렬이 쉬워짐
- toString()은 디버깅에 유용하게 만들기

## 🎓 학습 팁

### 효과적인 학습 방법
1. **작은 단위로 구현하고 테스트**
   - 한 번에 모든 것을 구현하려 하지 말기
   - 메서드 하나씩 구현하고 테스트

2. **테스트 주도 개발(TDD)**
   - 테스트 코드를 먼저 이해하기
   - 테스트를 통과시키는 것을 목표로 구현

3. **디버깅 활용**
   - System.out.println() 적극 활용
   - IDE의 디버거 사용법 익히기

4. **참고 자료**
   - Java Collections Framework 문서
   - Comparable과 Comparator 인터페이스
   - 불변 객체(Immutable Object) 패턴

### 주의사항
- 인터페이스를 변경하지 말고 구현체만 작성
- 테스트 코드는 수정하지 말고 통과시키기
- 예외 처리를 꼼꼼히 하기
